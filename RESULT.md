# Результаты анализа: тестовое задание по фронтенду (кратко)

## Что было найдено и улучшено

### Качество кода
- Крупный монолитный `App.vue` в исходной версии, смешивающий UI и бизнес‑логику.
- Использование `any[]` и отсутствие явных типов задач.
- Смешение состояний, логики, хелперов и констант в одном файле.
- Неиспользуемые участки кода (`userTasks`, дополнительные фильтры, неиспользуемые функции, debug‑логи).
- Код с таймерами и обработчиками событий, потенциально ведущий к утечкам.

### Архитектура
- Отсутствие централизованного управления состоянием, всё состояние в `App.vue`.
- Нет чёткого разделения на контейнерные и презентационные компоненты.
- Магические строки и inline‑конфигурация фильтров и начальных данных.
- Логика дат и статистики встроена прямо в компонент вместо выноса в хелперы.

### Производительность
- Вся логика и UI собраны в одном компоненте без code splitting.
- Повторяющееся форматирование дат без кеширования.
- Интервалы без централизованной очистки.
- Использование тега изображения без учёта доступности и производительности в шапке.
- **Асинхронная загрузка компонентов**: `TaskStatsCard` и `TaskChangelog` подключаются через `defineAsyncComponent` для code splitting.
- **Оптимизация загрузки шрифтов**: вместо `unplugin-fonts` используются прямые импорты `@fontsource/roboto` (веса 400, 500, 700) для лучшего tree‑shaking и контроля загрузки.
- **Оптимизация сборки**: в Vite добавлены `manualChunks`, чтобы вынести `vuetify`, `vue` и `pinia` в отдельные vendor‑чанки для лучшего кэширования и параллельной загрузки.

### Безопасность
- **Потенциальная XSS‑уязвимость**: Загрузка SVG-файла через тег <object> с внешнего источника может создать риск выполнения вредоносного JavaScript (XSS). Поэтому вместо <object> используется более безопасный вариант — тег <img>.
- Безопасность приложения зависит от безопасности браузера и `localStorage`.
- Для данного типа приложения (клиентское управление задачами без серверной части) дополнительные меры безопасности не требуются.
- Vue автоматически экранирует пользовательский ввод в шаблонах, что предотвращает XSS‑атаки.

### SEO и доступность
- **Мета‑описание**: добавлен `<meta name="description">` в `index.html` для улучшения SEO.
- **robots.txt**: создан файл `public/robots.txt` с разрешением для всех поисковых роботов.
- **ARIA‑атрибуты**: добавлены `aria-labelledby` и `id` в `TaskList.vue` для связи элементов и улучшения поддержки screen readers.
- **ARIA‑лейблы**: добавлен `aria-label` для кнопки удаления задачи, улучшающий доступность для пользователей скринридеров.

### Новые добавленные возможности
- **Лента изменений (task changelog / activity feed)**, в которой отображаются:
  - какое действие произошло (создание / завершение / возобновление / удаление),
  - какая задача затронута (укороченный заголовок),
  - когда это произошло (дата и время).
- **Локальное хранение данных**, благодаря которому:
  - список задач,
  - лента изменений,
  - текущий выбранный фильтр
  сохраняются после перезагрузки страницы и перезапуска браузера.
